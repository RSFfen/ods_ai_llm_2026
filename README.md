## 3. Проблема

**Какую проблему решаете?**

В проекте пока собираюсь решить свою личную проблему - повысить скорость подготовки учебных материалов для лекций и лабораторных работ для своих студентов, которых обучаю по собственным оригинальным (поскольку готовых, удовлетворяющи меня, нет) курсам. Сейчас процесс в основном происходит так: нужный материал берется либо из опрелеленных мест (сайтов, соцсетей) интернета, либо из электронных книг/докуменов (при необходимости применяется OCR). Все найденное/обработанное сохраняется в файлах в формате ipynb, из которых затем делаются тексты лекций/лабораторных работ, презентации, тесты/упражнения.

**Масштаб проблемы**

Наверно, точно так же готовятся к занятиям почти все преподаватели вузов, когда преподают новый курс собственной разработки. (Конечно, если курс уже прочитан хотя бы один раз, затрат на его подготовку становится гораздо меньше, нужно только следить за новостями по тематике курса и улучшать имеющийся материал на основе его апробирования на студентах.)  К счастью, у меня пока немного занятий - 3 пары раз в 2 недели, но подготовка к этим 4,5 часам все равно занимает все время (наверно, по закону Паркинсона "Работа занимает все возможное для нее время" Ж(:) Хотелсь бы ускорить этот процесс подготовки, чтобы освободить больше времени на другие занятия, например, на участие в проекте по курсу "LLM: От понимания к продукту" :)
 
## 4. Решение

**Что предлагается?** 

Есть идея побольше переложить функций в на целом на ИТ, и на ИИ, в частности. Например, после того, как необходимые материалы собраны (а собирать пока я собираюсь делать это самостоятельно, не полностью полагаясь на ИИ, потому что считаю, что отбор материала - именно то ценное в моих курсах, это мой авторский взгляд), "натравить" на эти материалы какую-нибудь хорошую LLM, чтобы она сделала работу по созданию разделов курса, саммаризации материалов в каждом разделе, подготовила презентацию и тесты/вопросы для каждого занятия в соответствии с имеющимися материалами. Хочется подчеркнуть - LLM должна работать только с теми материалами, которые ей предоставлены, ни малейшего материала "извне" в выдаче быть не должно. (Если же что-то дополнительно все-таки понадобится, пусть LLM сообщит об этом мне, "хозяину" входных данных.) 

**Что делает LLM в таком решении?**

Очевидно, LLM в этом случае выполняет вспомогательную роль, работает полностью под моим контролем. Более того, она не должна быть "шибко умной", быть просто хорошим помощником в обработке существующих текстов. Причем каждая составленная ею фраза должна иметь ссылку на источник, откуда это взято, с точностью чуть ли не до конкретного параграфа, с тем, чтобы этом можно было легко и быстро проверить. Не помешает также опция "рассудительности", где она будет разъяснять, как пришла к тому или другому выводу.
 
**Технический подход**

Конечно, основной подход - это использование RAG и open-source моделей (не самых "навороченных"), ollama + python. В качестве векторной базы - qdrant. В качестве UI-интерфейса - Streamlit или Django. Каждый из этих элементов уже апробирован по отдельности, хотелось бы, чтобы проект помог собрать все это в одну общую систему.
